L1 Market Data Ingestion

Polygon WebSocket (equities) + Binance REST (crypto) → 1m OHLCV bars + features

PolygonWebSocketAgent: Subscribes to Q(Quote), T(Trade), A(Aggregate) events for WATCHLIST symbols. Writes to market_data_l1, market_data_l2, order_flow tables.

BinanceRestAgent: Polls trades (5s), depth (2s), klines (60s). REST fallback for WebSocket-blocked environments.

FeatureAgent: Computes 20 features per bar: RSI(14), MACD, Bollinger Bands, VWAP, EMA spread, relative volume, trend strength, volatility regime, etc. Writes to features table (long format) and features_wide (materialized view).

SCOUT Scout Network Intelligence

12+ scouts continuously monitor regime, liquidity, correlation, news, social media

RegimeScout (60s): Classifies volatility (low/normal/high/panic), trend (trending_up/down/choppy), liquidity (deep_liquid/normal/thin/dangerous), compression/expansion patterns.

LiquidityScout (120s): Measures spread (bps), depth imbalance, slippage risk. Falls back to volume-based heuristics when L2 data unavailable.

CorrelationScout (300s): Rolling pairwise correlations, panic correlation detection, regime break identification.

External Scouts: News RSS (30min), Reddit WSB (60min), Discord (60min), YouTube finance (2h), Podcast (4h) → sentiment + hypothesis scores.

ALL feeds consumed by ScoutSynthesisEngine → confidence-weighted market narrative.

L2 Strategy Ideation

5 parallel Ideator Agents + IdeatorAgentV2 grammar path generate trading strategies via Claude LLM

V1 (5 agents): Each agent specializes in (asset × archetype) pair. Builds rich context (live features, failed patterns, successful patterns, blacklist), sends to Claude Sonnet (3000 token budget, chain-of-thought). Falls back to local diverse templates if LLM fails (circuit breaker pattern).

V2 (Grammar path): Dual-path architecture: deterministic grammar templates + LLM refinement. Builds feature distributions, threshold memory from winning strategies, and regime-weighted rankings. Grammar templates guarantee signal generation.

Strategy Normalizer: Validates feature names (whitelist), rejects raw price thresholds, auto-rounds overprecise numbers, enforces max 4 conditions, deduplicates via MD5 signature, computes diversity score.

L2 Code Generation

CoderAgent converts normalized strategy specs into executable Python with compile() validation

Generates a Python class with generate_signals(df) → pd.Series method. Includes:
• Regime classification logic (volatility + trend + direction + Bollinger → composite regime)
• Position state machine: FLAT → ENTRY(1) → LONG(1) → EXIT(-1) → FLAT(0)
• Hold time constraints (MIN_HOLD_BARS, MAX_HOLD_BARS, cooldown)
• Regime gate: only enters trades when current regime ∈ VALID_REGIMES
• compile() validation before DB save (code_failed if syntax error)
• Code sanitizer removes unsafe patterns

Status: pending_code → pending_backtest

L3 Backtesting Engine

BacktestRunner exec()s generated code, runs on real market data with dynamic slippage model

Data Loading: Fetches market_data_l1 + features_wide for symbol with most bars. Filters equities to market hours (9:30-16:00 ET).

Signal Generation: Calls generate_signals(df) → signal series. If zero entries, applies momentum fallback.

Dynamic Slippage: vol_mult × volume_mult → clipped [0.5x, 3.0x]. Higher volatility + lower volume = wider spreads.

Scoring:
• Short Window (<20k bars): composite = 30% return + 25% PF + 20% WR + 15% DD + 10% trades
• Institutional (>20k bars): Full annualized Sharpe, Sortino, Calmar
• Regime Score: Count distinct regimes at entry points (multi-regime = robust)

Status: pending_backtest → pending_validation

L3 Validation & Tier Assignment

ValidatorAgent scores strategies and assigns tiers: elite, validated, research_candidate, repair_candidate, or failed

Phase 1 — Structural Sanity: Entry count ≥ 2, trade count ≥ 2, no entry/exit saturation.

Phase 2 — Performance Tests: Composite score ≥ 10 (dev)/20 (prod), drawdown > -80%, win rate ≥ 0.05/0.15, profit factor ≥ 0.05/0.30.

Phase 3 — Cost Governance: Edge per trade > frequency-adjusted round-trip cost. Cost trap detection for high-frequency strategies.

Phase 4 — Tier Assignment:
Dev: elite≥60, validated≥35, research≥25, repair≥15
Prod: elite≥90, validated≥70, research≥50, repair≥30

Scout-Aware: Thresholds adjusted by liquidity/execution regime (dangerous → ×0.7 stricter).

L2 Evolutionary Engine (Mutation Loop)

MutatorAgent + CombinerAgent create offspring through deterministic micro-mutations and Claude-guided evolution

MutatorAgent (every 5 min):
• Tournament selects 5 from 30 candidates (tournament_size=7)
• 7-12 deterministic mutations: threshold relaxation (×0.8), tightening (×1.2), condition removal, RSI shift (+5), hold time ±5, cooldown ±3, regime filter adjust
• 1 Claude mutation (conservative, 1-3 changes only)
• Viability pre-screening ≥ 0.15, anti-clone (Jaccard distance > 0.05)
• Mutation families: repair, refinement, exploration, aggression, simplification
• Cost efficiency delta tracking per mutation

CombinerAgent (every 2h):
• Tournament-select 2 unique parents from top-30 pool
• Claude creates hybrid offspring combining best entry/exit logic
• Records combination lineage (parent_a + parent_b → child)

L5 Execution Gateway

The SOLE approved execution path: Signal → Idempotency → Kill Switch → Risk → Broker → Position

Execution Pipeline:
1. Idempotency gate (Redis processed set)
2. Distributed lock (Redis SET NX with TTL lease)

4. Scout-adaptive sizing: thin → 50% cut, dangerous → 75% cut

  tournament = random.sample(
    candidates, tournament_size
  )
  winner = max(tournament,
    key=fitness_score
  )
  selected.append(winner)

slippage = clip(combined, 0.5, 3.0)
