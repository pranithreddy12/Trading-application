# ATLAS Execution Flow

**Strategy Lifecycle from Ideation to Retirement**

---

## 1. Overview

ATLAS operates a continuous strategy lifecycle — strategies are generated, tested, deployed, monitored, and retired autonomously. The flow is orchestrated by the `MetaOrchestrator` and spans all seven agent layers.

```
IDEA ──► CODE ──► BACKTEST ──► VALIDATE ──► DEPLOY ──► MONITOR ──► RETIRE
│          │          │            │             │            │
└─ L2 ─────┘          │            │             │            │
                  └─ L3 ───────────┘             │            │
                                             └─ L4/L5/L6 ────┘
                                                           └─ L7 ──┘
```

## 2. Strategy Specification (L2 — IdeatorAgentV2)

### 2.1 Trigger

The IdeatorAgentV2 runs in a continuous loop:

```python
async def run(self):
    while not self._stop_requested:
        ctx = await self._build_context()       # Gather intelligence
        spec = await self._generate(ctx)         # LLM generates spec
        spec = await self._check_diversity(spec) # Governance gate
        spec = await self._check_economics(spec) # Economic constraints
        await self._publish(spec)                # Send to coder
        await asyncio.sleep(sleep_interval)
```

### 2.2 Context Building

Before generating a strategy, the ideator gathers:

**Market Context:**
- Current regime (from scout synthesis)
- Asset class (equity / crypto)
- Recent strategy performance (sharpe, win rate, profit factor)

**Scout Intelligence:**
- News sentiment scores
- Hypothesis validation results
- Source reliability weights

**Ecological Pressure:**
- Active population count
- Trade throughput (24h)
- Mutation diversity
- Portfolio participants
- Generation budget computation

```python
# Ecological pressure adjusts generation intensity:
ecological_pressure = compute_pressure(validated_count, active_count, trade_throughput)
generation_budget = max(1, int(ecological_pressure * 2))
```

**Diversity Governance:**
- Recent feature combos (last 50 strategies)
- Feature family distribution (oscillators, trend, volatility, volume)
- Adaptive similarity thresholds based on regime and throughput

The ideator constructs a structured prompt that includes:


## 3. Code Generation (L2 — CoderAgent)

### 3.1 Process

The CoderAgent converts a specification into executable Python:

```python
class GeneratedStrategy:
    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        # Strategy-specific logic generated by LLM
        # Conditions injected from spec
        # Regime classification auto-generated
        signals = pd.Series(0, index=df.index)
        # ... signal logic ...
        return signals
```

### 3.2 Structure

Every generated strategy includes:

1. **Imports** — Standard library + numpy + pandas
2. **Regime Classification** — Automatically generated code to classify bars
3. **Entry Conditions** — From spec, with conjunction logic
4. **Exit Conditions** — From spec, with conjunction logic
5. **Position State Machine** — Prevents exit spam, enforces holding
6. **Regime Gate** — Skip entries if regime not in valid set
7. **Return** — `pd.Series` of signals (-1, 0, 1)

### 3.3 Naming Convention

```
{archetype}_{ticker}_{primary_feature}_{4-digit timestamp}
Example: momentum_nvda_ema_spread_9341
```

This ensures globally unique names for database tracking.

### 4.2 Dynamic Slippage Model

Slippage is computed per-bar based on market conditions:

```python
def _compute_dynamic_slippage(self, df: pd.DataFrame) -> np.ndarray:
    vol = df["rolling_volatility"].astype(float).values
    rel_vol = df["relative_volume"].astype(float).values

    # Volatility multiplier: normalized by median
    vol_multiplier = np.clip(vol / max(median_vol, 1e-8), 0.5, 3.0)

    # Liquidity multiplier: lower volume → higher slippage
    liq_multiplier = np.clip(1.0 / np.maximum(rel_vol, 0.1), 0.5, 2.0)

    return BASE_SLIPPAGE * vol_multiplier * liq_multiplier
```

**Base slippage:** 3 bps  
**Range:** ~1.5 bps (calm, liquid) to ~18 bps (volatile, illiquid)

### 4.3 Commission Model

### 4.4 Metrics Computation

Institutional mode computes annualized metrics:

```python
bars_per_year = 525600  # Minute bars
sharpe_ratio = np.sqrt(bars_per_year) * (mean_return / std_return)
cagr = (1 + total_return) ** (bars_per_year / n_bars) - 1
max_drawdown = min(cum_return / cum_return.cummax() - 1)
sortino_ratio = sqrt(bars_per_year) * (mean_return / downside_std)
profit_factor = gross_profit / abs(gross_loss)
```

**Minimum trade threshold:** 2 trades (Phase 30, reduced from 5 for higher density)

### 4.5 Train/Holdout Split

- **Training window:** 60% of data (first 36 minutes of 60-min window)
- **Holdout window:** 20% of data (middle 12 minutes)
- **Validation window:** 20% of data (last 12 minutes)
- Walk-forward analysis measures sharpe decay from train → holdout

## 5. Validation (L3 — ValidatorAgent)

### 5.1 Institutional Rules (Production)

| Metric | Threshold | Rationale |
|---|---|---|
| Sharpe Ratio | ≥ 1.0 | Risk-adjusted return minimum |
| Max Drawdown | ≥ -25% | Capital preservation |
| Minimum Trades | ≥ 30 | Statistical significance |
| Win Rate | ≥ 45% | Trade quality |
| Profit Factor | ≥ 1.2 | Gross profit vs gross loss |
| Overfit Ratio | ≤ 50% | Holdout sharpe ≥ 50% of train sharpe |

## 8. Execution (L5 — ExecutionGateway)

### 8.1 Paper Trading Flow

```python
async def execute(self, strategy, allocation):
    signals = strategy.generate_signals(current_market_data)
    orders = self._position_sizing(signals, allocation)
    deduped = self._deduplicate(orders)

    for order in deduped:
        result = await self._simulate_execution(order)
        await self._log_trade(result)
        await self._broadcast_execution(result)
```

### 8.2 Execution Realism

Simulates real-world execution constraints:

```python
class ExecutionRealism:
    async def simulate(self, order):
        fill_prob = self._calculate_fill_probability(market_conditions)
        slippage = self._estimate_slippage(order.size, liquidity)
        latency = self._estimate_latency(network_conditions)

        return SimulationResult(
            filled=random.random() < fill_prob,
            price=order.price * (1 + slippage),
            latency_ms=latency
        )
```

### 9.3 Hypothesis Engine

Maintains a registry of beliefs with evidence tracking:

```python
class HypothesisEngine:
    async def evaluate(self, hypothesis):
        evidence = await self._gather_evidence(hypothesis)
        contradictions = await self._gather_contradictions(hypothesis)
        hypothesis.confidence = evidence / (evidence + contradictions)
        hypothesis.decay_rate = self._compute_decay(hypothesis.age)
        return hypothesis
```

### 9.4 Strategy Retirement

```python
class StrategyRetirementEngine:
    async def scan(self):
        candidates = await self._find_underperformers()
        for strategy in candidates:
            reasons = []
            if strategy.drawdown > MAX_DRAWDOWN:
                reasons.append("excessive_drawdown")
            if strategy.sharpe_90d < 0.3:
                reasons.append("sharpe_decay")
            if strategy.age > MAX_AGE_DAYS:
                reasons.append("age_limit")
            if strategy.drift_score > DRIFT_THRESHOLD:
                reasons.append("drift_exceeded")
            await self._retire(strategy, reasons)
```
